# V16 LLM-variant retriever term-map SFT

## Hypothesis

V16 should improve term-map adoption with less distribution shift than V15.  It
replaces selected canonical GT target translations with natural LLM-generated
variants in both the current `term_map` and the future assistant reference, so
the Speech LLM must use the provided map more often while avoiding artificial
marker suffixes.

## Background / Motivation

V13 used inference-aligned retriever timeline term maps but did not improve
tagged ACL quick eval.  V15 added explicit marker suffixes.  V16 keeps the same
V13 retrieval distribution but uses short natural Chinese target variants
generated by OpenAI and cached in the data-prep output.

## Expected metrics

Primary downstream check is tagged ACL quick eval on `zh lm=2 raw`, focusing on
`TERM_ACC`, `REAL_ADOPT`, `TERM_FCR`, BLEU, and StreamLAAL.  Compared with V15,
V16 should be less likely to hurt BLEU if natural variants are fluent.
