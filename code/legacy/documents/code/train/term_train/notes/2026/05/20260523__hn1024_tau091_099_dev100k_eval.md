# HN1024 tau 0.91..0.99 dev100k PR extension

## Hypothesis

The HN1024 dev gs100k PR curve should be extended beyond tau `0.90` so the
precision axis can reach the high-precision region near `30` without relying on
visual extrapolation.

## Background / Motivation

The current dev-only PR figure is cropped to the matched-precision region.  This
looks cleaner than the original long-tail plot, but it hides whether no-HN
overtakes HN1024 after recall drops further.  The existing HN1024 sweep
`31xmxmdp` stops at tau `0.90`, where dev gs100k precision is only about
`28.19`.

## What changed vs baseline

- Checkpoint remains `lh1b88kw` best-secondary.
- Eval scope is dev-only, gs100k only.
- Held-out ACL, tagged ACL, and medicine are disabled.
- Tau grid is `0.91 0.92 0.93 0.94 0.95 0.96 0.97 0.98 0.99`.
- The result is only used to extend the dev PR curve; it is not a new
  downstream calibration rule by itself.

## Expected metrics

The run should add HN1024 dev gs100k filtered precision/recall points beyond
the current precision `28.19` endpoint.

## Verdict

SUCCESS.  W&B run: `ifs45d6j`.

moderate precision region, while the far-right tail is too recall-expensive.
