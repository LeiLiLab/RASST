## Main Finding

The plotted medicine offline+GT lines and RASST points are not denominator-matched.

This is not because the source/audio test examples are different. The source
text files used by the RASST/no-RAG medicine prep and the no-RAG baseline
medicine prep are byte-identical and both contain 1437 source sentences. The
difference is on the target-reference side:

- RASST/no-RAG medicine inputs are generated by
  `documents/code/simuleval/prepare_medicine_one_talk_inputs.py`, which reads
  `sent["translations"][lang]` from
  `/home/jiaxingxu/rag-sst/eso-dataset/outputs_v2/test/sample_*_v2/sentences_v2.json`.
- Siqi's offline outputs carry references inside
  `/mnt/data/siqiouyang/runs/infinisst_rag/offline/eso/*/*/instances.log`,
  with sentence-level audio paths under
  `/data/group_data/li_lab/siqiouya/datasets/eso_dataset/segmented_wavs/...`.
- The hard glossary was generated against the restored/final target-reference
  line, not the older `outputs_v2/test` line. In particular,
  `outputs_v2_abbrev_exact_match_abbrev_restored_source_matched/test` and
  `outputs_v2_final/test` have identical target references, and Siqi's offline
  `instances.log` references match `outputs_v2_final/test` exactly.

Thus the same ESO talks/sentence segmentation are being used, but not exactly
the same target translation text.

Reference-version diff counts across the five medicine samples:

| lang | `outputs_v2/test` vs `outputs_v2_final/test` | `outputs_v2_final/test` vs Siqi offline |

So the version mismatch is specifically: RASST/no-RAG used old
`outputs_v2/test`, while the hard glossary and offline upper-reference rows are

## Why TERM_TOTAL Exceeds Glossary sentence_ids

The normalized hard-medicine glossary file

`/mnt/gemini/home/jiaxuanluo/medicine_eval_hard_terms_llm_judge_manual_20260524/hard_medicine_glossary_raw_llm_judge_manual_zh215_unique212.json`

contains 212 unique term entries. Before the 2026-05-25 sentence-id backfill,
summing the stored `sentence_ids` lists gave only 280 manually checked evidence
occurrences. That evidence-only version was insufficient for offline oracle
term-map construction because many later source occurrences of the selected hard
terms were not labelled.

The current glossary has been backfilled from
`/home/jiaxingxu/rag-sst/eso-dataset/outputs_v2_final/test`: the original
evidence ids are preserved as `manual_evidence_sample_ids` and
`manual_evidence_sentence_ids`, while the compatibility `sample_ids` and
`sentence_ids` fields now describe full source-side occurrences. Use
`sample_sentence_ids` for exact paired ids, because medicine `sentence_id` is
only unique within each sample. The paired source occurrence count is now 869.
The language-specific `lang_sample_sentence_ids` fields are target-deduplicated
to match the TERM_ACC scorer: zh 744, de 718, ja 736. The pre-dedup language
fields are preserved under `*_before_target_dedup`.

These sentence-id fields support offline oracle term-map construction; they are
now aligned to the current TERM_ACC scorer for language-specific offline oracle
term-map construction.

The SimulEval scorer loads the glossary by target translation and then scans
every resegmented source/reference sentence pair. A term contributes to
`TERM_TOTAL` whenever the source term occurs in the aligned English source and
the target translation occurs in the aligned target reference. The main-result
offline mirror uses the same rule. Therefore a term selected from one manually
checked hard occurrence can contribute many denominator items if it appears
elsewhere in the 1437-sentence ESO test set.

With the current scorer and final/offline references, this hard glossary gives:

| lang | TERM_TOTAL | unique target terms with at least one counted occurrence | top counted source terms |
| --- | ---: | ---: | --- |
| zh | 744 | 210 | `dose` 121, `radiotherapy` 45, `HER2` 31 |
| de | 718 | 204 | `dose` 123, `radiotherapy` 43, `HER2` 31 |
| ja | 736 | 210 | `dose` 116, `HER2` 31, `rectal cancer` 22 |

So the 600--700 denominators are expected under the current "global hard
glossary over all matching reference occurrences" metric. If the intended
benchmark is exactly the manually judged hard occurrences, the scorer should be
changed to restrict counting to the glossary's `(sample_id, sentence_id, term)`
evidence set or to a derived per-sentence eval map.

## Conclusion

Do not interpret the current medicine offline+GT horizontal line as a strict upper bound over the RASST points. The RASST and offline+GT rows are derived from different target-reference variants, and the de RASST rows are already marked `dirty_untrusted`.

Before using this comparison in the paper, either:

1. re-evaluate offline+GT with the exact RASST/no-RAG medicine reference files and auditable oracle term-map inputs, or
2. remove/mark the medicine offline+GT horizontal line as a non-comparable reference, not an upper bound.
